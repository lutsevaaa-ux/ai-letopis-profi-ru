# Шаг 1. Зафиксировать пилотный срез данных

## Что именно мы запускаем

Для пилота берем три канала Mattermost:

- `Инфочат-поддержка`
- `random` в команде `profi`
- `general` в команде `profi`

Период обработки:

- c `2026-02-01` по `2026-05-01`
- рабочее правило диапазона: начало включительно, конец не включительно
- это значит, что в пилот попадают все сообщения за февраль, март и апрель 2026 года

## Что считаем кандидатом в событие

Базовые сигналы:

- у поста есть активное обсуждение: не меньше `5` сообщений в треде
- у поста есть заметный отклик по эмодзи-реакциям

На текущем шаге зафиксировано важное ограничение: в проверенных ответах MCP реакции пока не видны отдельным полем. Поэтому для следующего шага логика отбора будет строиться так:

- обязательный сигнал на старте: размер треда
- сигнал по реакциям: проверяем дополнительно, можно ли его достать текущими инструментами
- если реакции недоступны, начинаем с тредов и затем добавляем реакции позже

## Что должно получиться на выходе пилота

Нужны `5-10` карточек-кандидатов, где для каждой есть:

- дата
- канал
- тема или рабочий заголовок
- короткое обезличенное описание
- идентификатор исходного поста
- ссылка на исходный пост
- размер треда
- идея для мема

## Ограничения по данным

Персональные данные не используем.

Правило обработки:

- не выносим в итог имена и логины
- не копируем чувствительные детали дословно
- если тред полезный, но содержит личные данные, делаем нейтральный пересказ сути

## Что уже подтверждено технически

Через текущий Mattermost MCP уже подтверждено следующее:

- каналы доступны
- поиск сообщений по команде, каналу и датам работает
- треды по `postId` читаются

Подтвержденные идентификаторы каналов:

- `Инфочат-поддержка`: `akacewjx5iytmgukdan4rnbkyw`
- `random`: `miofoz959jre5b1ppk56doeddw`
- `general`: `rjmqpp9getd7znjfeep7o9kydh`

## Что будет следующим шагом

Следующий рабочий шаг: сделать первую выгрузку сообщений и тредов за этот период, чтобы собрать черновой список кандидатов событий и проверить, хватает ли нам текущего MCP без дополнительной интеграции.
