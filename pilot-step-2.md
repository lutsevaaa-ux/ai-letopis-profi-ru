# Шаг 2. Первая выгрузка кандидатов событий

## Что сделано

Собран первый проход по пилотным каналам за период `2026-02-01` — `2026-05-01`.

В результате подготовлен список кандидатов в события на основе:

- размера треда
- содержательной значимости темы
- пригодности темы для будущей карточки летописи

## Что важно

На этом проходе мы **не использовали эмодзи-реакции как сигнал отбора**, потому что текущий Mattermost MCP пока не отдал реакции отдельным полем.

Из-за этого логика была такой:

- сначала смотрим на треды с `5+` сообщениями
- затем вручную убираем слабые или слишком локальные темы
- оставляем те, которые выглядят как хорошие карточки для летописи

## Что получилось

Первый рабочий набор кандидатов сохранён в:

- `pilot-candidates-step-2.json`

В нём сейчас `8` карточек-кандидатов.

Они делятся на два типа:

- продуктовые и операционные изменения
- культурные и мемные внутрикомандные эпизоды

## Сильные кандидаты первого прохода

Самые заметные на текущем этапе:

- запуск первых сценариев `Профи-Стандарта`
- большой тред про зарубежную карту и `eSIM` для доступа к зарубежным сервисам
- упрощение правил по газовому оборудованию
- обсуждение внешней рекламы конкурента как зеркала для собственного продукта

## Ограничения текущей выборки

- реакции по эмодзи пока не учтены
- часть тредов в `random` содержит пустые ответы, которые, вероятно, были медиа-постами или мемами без текста
- один из самых больших тредов продолжился уже после `2026-05-01`, поэтому для него в JSON сохранена нижняя оценка активности внутри пилотного периода

## Что делать дальше

Следующий логичный шаг:

- отранжировать эти `8` кандидатов
- выбрать из них `5-6` лучших
- привести их к единому формату карточек: месяц, тема, описание, мем-идея

После этого можно будет переходить либо к финальному текстовому черновику летописи, либо сразу к визуальной сборке.
